## 💡 My Philosophy

> The best automation starts with one simple question:  
> **Why am I still doing this manually?**

Laziness is often the starting point of progress.

I believe repetitive work should be handled by software wherever possible, leaving people free to solve meaningful problems, improve systems, build better tools, and focus on what actually requires human judgment.

I also believe the future of software development will not depend on choosing one “best” AI model.

It will depend on learning how to combine specialized agents, development tools, and human experience into one reliable workflow.

## 🇺🇦 UA Passport Slot Monitor

An open-source project designed to monitor appointment availability at Ukrainian passport service centres outside Ukraine.

The project aims to help Ukrainians abroad detect newly available appointment slots and receive timely notifications.

### Planned capabilities

- Monitoring multiple service centres and providers
- Configurable country and location selection
- Email, push, and messenger notifications
- Modular provider architecture
- Privacy-oriented data processing
- Support for manual CAPTCHA interaction where required
- Multilingual user documentation
- Extensible monitoring and notification components

**Current stage:** Research, architecture, documentation, and early development.

**Technologies and concepts:**

Python • Web Monitoring • Notifications • Provider Architecture • Privacy • AI-Assisted Development

## ⏱️ Devflow Tracker

An open-source time and activity tracking project focused on modern AI-assisted development workflows.

Traditional time trackers usually measure only time spent inside an editor. Devflow Tracker explores a broader approach that also considers research, AI conversations, architecture, testing, documentation, and work performed across multiple devices.

### The project is designed to track

- AI conversations and brainstorming
- Architecture and technical planning
- Coding and debugging sessions
- Testing and validation
- GitHub activity
- Documentation work
- Desktop and mobile development sessions
- Work distributed between humans and AI agents
- Multiple projects, applications, and development tools

The project explores how developer productivity can be measured when development work is shared between a human and several AI systems.

**Technologies and concepts:**

Python • Git • GitHub • AI Workflows • Developer Analytics • Activity Tracking • Multi-Agent Development

🔗 [View repository](https://github.com/DorianGray-web/Devflow-tracker)
